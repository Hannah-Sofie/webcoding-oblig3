Reflections

The preparations I did before structuring my html page was first to go through the sustainable development goals and pick one I thought was interesting and wanted to promote. I ended up picking goal 14, life below water, which was about conserving and sustainably use of the world's ocean, seas and marine resources.

After selecting goal 14, I chose target group #2, which was lower secondary education, age 14-16 (years 8-10).

When structuring and designing my page I used a number of images as a background, because I think that young people like things that are visual, colorful and fun. Images are not the most sustainable on web, but I used images in webp format, which significantly reduces the size of the image file compared to a jpeg file. So even if it is more sustainable not to use images, you cannot avoid using images on your website either. But if you are going to use images, you can at least try to make it more sustainable and reduce the size, as you don't always need a picture in 4K.

The color theme on my website is blue, as goal 14 is about the sea, and the color blue is often associated with the sea. I have therefore also made my footer so that it looks like a wave.

I tried to keep my site quite simple and intuitive considering that there are 14-16 year olds who will read and click through it. So to keep it clean and simple, there is a text box on each image page when scrolling. This is so that the children are not overwhelmed by too much text and information. It is best to keep it short and concise.

When it comes to choosing a font, I chose the same font that the UN Sustainable Development Goal page has on its website, roboto. It is a simple and clear font that is easy to read. All information, facts and texts in the text boxes are also taken from the UN Sustainable Development website.

The game I have created is a simple and fun bingo game that can suit most people. Both children and adults, so the children can both play with friends and family. Of course, if you have smaller children, an adult can help if, for example, they want to repair old clothes with holes in them. Or the children can join in the cooking if they are going to make a meat-free meal. So the tasks on the bingo table can be adapted to age, so that everyone in the family can join.

I have tried throughout my page to keep the same theme, with an exciting background and information/text box above. The only thing is my bingo print page, which is quite plain. This so that it will be easy to print the bingo table.

On the mobile version, I have used an icon for the menu button from fontawesome, which "collects and hides" all the nav links. This makes it easier for the mobile user as there is not a lot of text in the navigation menu that is squeezed together.

The bingo-table-game was first created with <table>, but because it is hard to make responsive, I made it to a grid. But it was still overflowing on smaller screens, so to solve this I had to remove the bingo table on the game page when the screen size came below 800px and put in on another page, so it could fit the whole screen without overflowing.

On the contact page, I have created a form where the person on the page can write and send us an email. They can choose what the message is about in the subject. It could be if they have questions, want to give feedback, have an idea or just want to share an experience, or something else.

In my code I have tried to keep it clean and structured. I have commented on the code, both to tell what the code does and to keep order in where the various codes belong together. The codes are validated and I have tried to make my page accessible. Among other things, I have put slightly transparent boxes behind the text so that it will be easier to read on such a colorful background. You can also hover over/press (on mobile) the text boxes to make them slightly larger and easier to read. I have not only done this to make it easier to read, but it is also a extra fun interaction the children can do with the boxes to keep their focus more on the page and the information in the text boxes.

When trying to WCAG validate my code, I get some problems with the <input type="checkbox" id="click"> and <i class="fas fa-bars">. I think this is because it misunderstands that it is supposed to be a button.

When CSS validating my code, I got "line 116 nav ul li a:hover and nav ul li a:hover, same color for background-color and color", which is pointed out for me as a warning, but this is just when you hover over the navigation links in the navigation menu, it switches colors. But there are no problem reading what it says, as i also switched the color from white to black, so it is readable on the white hovering background.

I have also tried to code efficiently, meaning I have created classes where I have been able to, and used the same code on them, so that I do not repeat myself with the same code on several of the <div>.

The site is responsive, and it has, at least, 2 different layout configurations: one for small screens (mobile phones, 600px) and another one for regular screens (laptops and desktop screens). When creating my site I first started with the desktop, even though I know it's good practice to start with the mobile one. But I couldn't figure out how to go from menu button to navigation menu with all the links aligned. It was easier to design the navigation menu, and then hide it in mobile. So that's why I designed the desktop before the mobile, but I know I should have started with the mobile.
